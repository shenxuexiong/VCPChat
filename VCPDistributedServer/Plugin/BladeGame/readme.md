## 华山论剑 小游戏 
* 这是一个回合制博弈类游戏。
  需要AI主动创建新游戏，才能开始游玩。
  创建游戏后，GUI被打开。
  人类用户需要输入回合动作，来进入“回合已准备”状态。只有当人类处于回合已准备状态时，AI的回合输入指令才能有效。否则应该提示人类决策未准备。
  因此，在这种情况下，ai和人类都不知道对方会输入何种指令。
  GUI：人类通过点击界面上的技能按钮来输入回合指令。当ai也输入指令时，当前回合被执行，并通过同步插件本身返回ai该回合游玩的结果。（包括历史回合流程。以及当前游戏进行状态。）
  
  游戏规则。
  初始所有AI和用户拥有5点生命值，0点剑气能量。生命值和能量上限是6点。
  
  游戏总共有如下操作
  
  蓄势(Charge) 获得一格剑气能量
  斩击(Slash) 消耗0能量。造成1点伤害。
  轻霜踏雪(LightStep) 消耗1能量。造成2点伤害。
  寒梅逐鹿(PlumBlossom) 消耗2能量。造成4点伤害并恢复1生命。
  回光无影(Flash) 消耗3能量。造成9点伤害。
  御剑格挡(Block) 消耗0能量 本回合受到伤害-4
  太极两仪(Taiji) 消耗0能量 本回合不会受到“回光无影”的伤害（只能防御这一个技能。）
  
  1.蓄势无法被打断，无论对方做什么，蓄势都能成功，但会受到相应伤害。
  2.当高级攻击遇到低级攻击时，低级攻击将会被打断，只有高级攻击生效，但若两边都是同等级攻击，则攻击抵消（寒梅逐鹿的恢复效果不会被打断）
  
开始游戏
<<<[TOOL_REQUEST]>>>
maid:「始」猫娘小克「末」，
tool_name:「始」BladeGame「末」,
command:「始」StartGame「末」
<<<[END_TOOL_REQUEST]>>>

进行游戏
<<<[TOOL_REQUEST]>>>
tool_name:「始」BladeGame「末」,
command:「始」PlayTurn「末」,
action:「始」Charge「末」,   //做出行为
reason:「始」我需要积攒剑气来释放大招。「末」 (可选)
<<<[END_TOOL_REQUEST]>>>
